Replicating Rarible's "Connect Wallet" experience involves a single, cohesive path using a modern front-end framework like React. The most effective stack for this is **Wagmi** combined with **RainbowKit**. This combination provides both the underlying logic for wallet interaction and a pre-built, elegant UI that handles a variety of wallets, including WalletConnect.

Here is a step-by-step breakdown of how to implement this on a React/Next.js application, with a focus on EVM-based chains.

### 1\. Installation and Configuration

Start by setting up your project and installing the necessary libraries. You will need a WalletConnect Project ID for WalletConnect functionality. You can get this for free from WalletConnect Cloud.

```bash
# In your project's root directory
npm install @rainbow-me/rainbowkit wagmi viem @tanstack/react-query
```

Next, configure Wagmi and RainbowKit with your desired chains. This configuration should be done in a central file, for example, `wagmi.ts`.

```typescript
// wagmi.ts
import { getDefaultConfig } from '@rainbow-me/rainbowkit';
import {
  mainnet,
  polygon,
  optimism,
  arbitrum,
  base,
} from 'wagmi/chains';

// Replace with your WalletConnect Project ID
const projectId = 'YOUR_WALLETCONNECT_PROJECT_ID';

const config = getDefaultConfig({
  appName: 'Coyn Messenger',
  projectId,
  chains: [
    mainnet,
    polygon,
    optimism,
    arbitrum,
    base,
    // Add any other EVM-based chains you need
  ],
  ssr: true, // Set to true if using Next.js with SSR
});

export default config;
```

-----

### 2\. Provider Setup

Wrap your entire application with the necessary providers to ensure that all components can access the wallet state. This is typically done in your `_app.tsx` or `layout.tsx` file in Next.js.

```typescript
// pages/_app.tsx or app/layout.tsx
import '@rainbow-me/rainbowkit/styles.css';
import { WagmiProvider } from 'wagmi';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { RainbowKitProvider } from '@rainbow-me/rainbowkit';
import config from '../wagmi';

const queryClient = new QueryClient();

function MyApp({ Component, pageProps }) {
  return (
    <WagmiProvider config={config}>
      <QueryClientProvider client={queryClient}>
        <RainbowKitProvider>
          <Component {...pageProps} />
        </RainbowKitProvider>
      </QueryClientProvider>
    </WagmiProvider>
  );
}

export default MyApp;
```

-----

### 3\. "Connect Wallet" UI Component

Now you can add the single, powerful `ConnectButton` component to your application. This button handles all the logic and UI, just like on Rarible.

```typescript
// components/Connect.tsx
import { ConnectButton } from '@rainbow-me/rainbowkit';

const Connect = () => {
  return (
    <div>
      <h1>Connect Your Wallet</h1>
      <ConnectButton />
    </div>
  );
};

export default Connect;
```

When you render this component, clicking the button will automatically display a modal with options for MetaMask, Coinbase Wallet, WalletConnect, and more.

-----

### 4\. Authentication with `useSignMessage`

The connection itself is not a secure login. You must implement a "Sign-In with Ethereum" (SIWE) flow. This involves getting the user to sign a message to prove they own the wallet.

Use Wagmi's `useSignMessage` hook to handle this on the front end.

```typescript
// components/SignIn.tsx
import { useSignMessage } from 'wagmi';
import { useAccount } from 'wagmi';
import { useState } from 'react';

const SignIn = () => {
  const { address, isConnected } = useAccount();
  const [nonce, setNonce] = useState('');
  const { signMessage, isLoading, data } = useSignMessage();

  // You would fetch a nonce from your backend here
  const fetchNonce = async () => {
    // const res = await fetch('/api/nonce');
    // const nonce = await res.json();
    setNonce('example-nonce-123'); // Example nonce
  };

  const handleSignIn = () => {
    if (!nonce) {
      fetchNonce();
      return;
    }
    signMessage({ message: `I am signing this message to prove ownership of this wallet.` });
  };

  return (
    <div>
      {isConnected && (
        <>
          <p>Connected: {address}</p>
          <button onClick={handleSignIn} disabled={isLoading}>
            {isLoading ? 'Signing...' : 'Sign In with Wallet'}
          </button>
          {data && <p>Signature: {data}</p>}
        </>
      )}
    </div>
  );
};
```

On your backend, you would then use a library like `siwe` to verify the signature against the wallet address and issue a session token.

This single path provides a complete and secure authentication flow that is both user-friendly and robust.
